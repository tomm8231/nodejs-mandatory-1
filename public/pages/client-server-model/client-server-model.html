Client-server Model

    
Client → Server: Klienten sender en HTTP-forespørgsel til serveren.
Client ← Server: Serveren svarer med en HTTP-response til clienten.

Klienten sender en forespørgsel til serveren. Serveren modtager og behandler forespørgslen, afgør hvad der bliver anmodet om, og sender en respons tilbage til klienten med de anmodede data eller bekræftelse af handlingen. Klienten modtager en respons og kan derefter vise eller bruge de modtagne data. Dette sker via et fetch-kald.

Client-server-modellen er en grundlæggende arkitektonisk komponent, der ligger til grund for kommunikationen mellem klienten og serveren. Den danner grundlaget for applikationer og muliggør effektiv dataudveksling og interaktion.
